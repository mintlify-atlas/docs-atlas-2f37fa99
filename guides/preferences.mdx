The NixOS Configuration Editor preferences allow you to customize how the application works and which configuration files it manages.

## Accessing preferences

You can open the preferences window from the application menu or when first launching the application.

<Steps>
  <Step title="Open the menu">
    Click the menu button in the application toolbar.
  </Step>
  
  <Step title="Select Preferences">
    Choose **Preferences** from the menu.
  </Step>
  
  <Step title="Adjust settings">
    Modify the settings as needed.
  </Step>
  
  <Step title="Close to save">
    Settings are automatically saved when you close the preferences window.
  </Step>
</Steps>

## Configuration file location

The most important preference is specifying which NixOS configuration file to edit.

### Selecting your configuration file

<CodeGroup>
```nix Default location
/etc/nixos/configuration.nix
```
</CodeGroup>

The configuration file setting is implemented in `src/ui/preferencespage.rs:57-88`:

```rust
adw::ActionRow {
    set_title: "Configuration file",
    add_suffix = &gtk::Box {
        gtk::Button {
            gtk::Label {
                set_label: {
                    let x = model.configpath.to_str().unwrap_or_default();
                    if x.is_empty() {
                        "(None)"
                    } else {
                        x
                    }
                }
            },
            connect_clicked[sender] => move |_| {
                sender.input(PreferencesPageMsg::Open);
            }
        },
    }
}
```

<Note>
The configuration file path is stored in the application's persistent configuration and loaded each time you start the application.
</Note>

### Custom configuration locations

You can specify any NixOS configuration file:

- Development configurations in your home directory
- Alternative configurations for testing
- Configuration files in version control repositories

<Tip>
If you manage multiple systems, you can switch between different configuration files by changing this setting.
</Tip>

## Flake support

NixOS flakes provide a modern way to manage system configurations. The editor supports both traditional and flake-based configurations.

### Enabling flakes

<Steps>
  <Step title="Toggle flake support">
    Enable the **Use nix flakes** switch in preferences.
  </Step>
  
  <Step title="Select your flake file">
    Click the **Flake file** button to choose your `flake.nix` file.
  </Step>
  
  <Step title="Specify flake arguments">
    Enter the flake reference in the **Flake arguments** field.
  </Step>
</Steps>

The flake switch implementation from `src/ui/preferencespage.rs:89-106`:

```rust
adw::ActionRow {
    set_title: "Use nix flakes",
    add_suffix = &gtk::Switch {
        connect_state_set[sender] => move |_, b| {
            if b {
                sender.input(PreferencesPageMsg::SetFlakePath(Some(PathBuf::new())));
            } else {
                sender.input(PreferencesPageMsg::SetFlakePath(None));
                sender.input(PreferencesPageMsg::SetFlakeArg(None));
            }
            gtk::Inhibit(false)
        },
    }
}
```

### Flake file selection

When flakes are enabled, you can select your flake file:

<CodeGroup>
```nix Typical flake location
/etc/nixos/flake.nix
```
</CodeGroup>

The file picker shows only the filename, but stores the full path:

```rust
// From src/ui/preferencespage.rs:124-137
gtk::Label {
    set_label: {
        let x = if let Some(f) = &model.flake {
            f.file_name().unwrap_or_default().to_str().unwrap_or_default()
        } else {
            ""
        };
        if x.is_empty() {
            "(None)"
        } else {
            x
        }
    }
}
```

### Flake arguments

The flake arguments specify which system configuration to build from your flake:

<CodeGroup>
```bash Format
--flake /path/to/flake.nix#<argument>
```

```bash Example
--flake /etc/nixos/flake.nix#my-hostname
```
</CodeGroup>

The argument field in `src/ui/preferencespage.rs:145-161`:

```rust
adw::EntryRow {
    set_title: "Flake arguments (--flake path/to/flake.nix#<THIS ENTRY>)",
    connect_changed[sender] => move |x| {
        sender.input(PreferencesPageMsg::SetFlakeArg({
            let text = x.text().to_string();
            if text.is_empty() {
                None
            } else {
                Some(text)
            }
        }));
    },
}
```

<Info>
The flake argument typically matches your system hostname, but can be any output defined in your flake.
</Info>

## Configuration persistence

Preferences are stored using the nix-data library and persisted between sessions.

### Configuration structure

The `NixDataConfig` structure (referenced in `src/parse/preferences.rs:1-16`) stores:

```rust
pub struct NixDataConfig {
    pub systemconfig: Option<String>,    // Path to configuration file
    pub flake: Option<String>,           // Path to flake file (if enabled)
    pub flakearg: Option<String>,        // Flake argument (if enabled)
    pub generations: Option<u32>,        // Number of generations to keep
}
```

### Loading configuration

Preferences are loaded when the application starts:

```rust
// From src/parse/preferences.rs:4-10
pub fn getconfig() -> Option<NixDataConfig> {
    if let Ok(c) = nix_data::config::configfile::getconfig() {
        Some(c)
    } else {
        None
    }
}
```

### Saving configuration

Changes are saved when you close the preferences window:

```rust
// From src/ui/preferencespage.rs:246-259
PreferencesPageMsg::Close => {
    if !self.configpath.eq(&self.origconfigpath)
        || !self.flake.eq(&self.origflake)
        || !self.flakearg.eq(&self.origflakearg)
        || self.error
    {
        let _ = sender.output(AppMsg::SetConfig(NixDataConfig {
            systemconfig: Some(self.configpath.to_string_lossy().to_string()),
            flake: self.flake.as_ref().map(|x| x.to_string_lossy().to_string()),
            flakearg: self.flakearg.clone(),
            generations: self.generations,
        }));
    }
}
```

<Warning>
If you point the editor at an invalid configuration file, you'll need to fix the path in preferences before you can use the application.
</Warning>

## First-run experience

When you first launch the application without saved preferences, you'll see a welcome screen prompting you to configure these settings:

1. The welcome screen appears
2. Click through to open preferences
3. Set your configuration file path
4. Optionally enable and configure flakes
5. Close preferences to start editing

<Tip>
The application validates your configuration file path when you save preferences. If the file doesn't exist or isn't readable, you'll see an error.
</Tip>

## Preference validation

The application performs several validation checks:

### Configuration file checks

- File must exist and be readable
- File must be a valid Nix configuration
- User must have permission to read the file

### Flake checks

- Flake file must exist if flakes are enabled
- Flake argument is required when using flakes
- Path must point to a valid `flake.nix` file

## Troubleshooting preferences

<AccordionGroup>
  <Accordion title="Configuration file not found">
    The path you specified doesn't exist or isn't accessible.
    
    **Solution**: Verify the path and ensure you have read permissions.
  </Accordion>
  
  <Accordion title="Flake validation failed">
    Your flake configuration is incomplete or invalid.
    
    **Solution**: Ensure both the flake file and flake argument are set correctly.
  </Accordion>
  
  <Accordion title="Preferences won't save">
    The application can't write to its configuration directory.
    
    **Solution**: Check permissions on `~/.config/nix-data/`.
  </Accordion>
</AccordionGroup>

## Advanced configuration

For advanced users, the preferences are stored in:

<CodeGroup>
```bash Configuration location
~/.config/nix-data/config.json
```
</CodeGroup>

You can manually edit this file if needed, though the application provides a UI for all common use cases.

<Note>
Changes to the configuration file while the application is running may not take effect until you restart the application.
</Note>
